## 2. Adapter matrix

Format: `auth | direction | status | tested?`
- **auth**: OAuth = OAuth2 with refresh-token flow; Key = static API key / HTTP Basic / Bearer; Mixed = OAuth + signed app secret; Webhook = also accepts inbound webhooks (signature-verified).
- **direction**: read / write / both.
- **status**: live = real HTTP to vendor; stub = `Not implemented` or returns hardcoded data; partial = some methods real, others stubbed.
- **tested?**: registry = covered only by `adapters.test.ts` registry checks; integration = has dedicated test file; mocked = has fixtures but no contract tests; live-verified = manually confirmed against a sandbox (not currently any).

## 4. Page matrix and shell-tab architecture

`client/src/App.tsx` only routes the **shells**: `Home`, `Architect`,
`Merchant`, `Social`, `Storefronts`, `Inbox`, `Insights`, `Settings`,
`Workflows`, `Onboarding`, `Landing`, `LegalPage`, `NotFound`,
`InviteAccept`, `StatusPage`, `PromptLab`, `Config`, `Chat`. The remaining
"pages" are **mounted as tabs inside shells**, which is intentional:

| Shell | Embedded tab pages |
| --- | --- |
| `Inbox.tsx` | `Activity.tsx`, `Approvals.tsx` |
| `Insights.tsx` | `Analytics.tsx`, `Intelligence.tsx`, `CampaignFunnel.tsx` |
| `Settings.tsx` | `Profile.tsx`, `Members.tsx`, `BotSettings.tsx`, `PlatformHealth.tsx` |
| `Storefronts.tsx` | `Integrations.tsx`, `PluginStore.tsx`, `SupplierPOs.tsx`, `GmailBot.tsx` |

So no page is truly orphaned, but Sprint-26 audit notes claiming
"all pages registered in App.tsx" are out of date — the model has moved to
shells with embedded tabs. Subsequent docs should describe this explicitly.
